<Update label="2024-09-15" description="v2.1.0" tags={["breaking", "feature"]}>

## Breaking Changes

- Updated API endpoints from `/v1/pages` to `/v2/pages`. Migrate by updating your requests.

<Steps>
  <Step title="Update Endpoint">
    Change `https://api.example.com/v1/pages/{pageId}` to `https://api.example.com/v2/pages/{pageId}`.
  </Step>
  <Step title="Test Integration">
    Verify your integration with the new `<ParamField path="pageId" param-type="string" required="true">` parameter.
  </Step>
</Steps>

## New Features

- Launched content optimization recommendations based on audience insights.

</Update>

<Expandable title="Detailed Migration Notes for v2.1.0" default-open="false">

If you encounter issues after upgrading:

```javascript
// Old way (v1)
const insights = await fetch('https://api.example.com/v1/pages/{pageId}?key=YOUR_API_KEY');

// New way (v2)
const insights = await fetch('https://api.example.com/v2/pages/{pageId}?key=YOUR_API_KEY');
```

Update your `{pageId}` parameter and test thoroughly.

</Expandable>
